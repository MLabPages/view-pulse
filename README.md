# ViewPulse

端末内の画像・動画またはYouTube動画を大きく表示しながら、内カメで表情とおおまかな視線を端末内解析するWebアプリです。

## 主な機能

- 端末内の画像または動画を選び、全画面に近い表示で閲覧
- YouTube URL、短縮URL、Shorts URLを公式埋め込みプレイヤーで再生
- YouTubeの再生時間と分析データを同期
- 対応端末では、インストールしたPWAをYouTubeの共有先として起動
- 内カメ1台とMediaPipe Face Landmarkerによる表情・顔向き・視線の端末内解析
- 動画は再生時間、画像は記録開始からの経過時間と分析データを同期
- 記録中の内カメを「小さなピクチャーインピクチャ」と「完全に隠して解析中バーだけ」から選択し、途中でも切替
- 5点の視線キャリブレーション
- コンテンツ上の視線ヒートマップ、反応タイムライン、注視率・笑顔率・最多注視領域
- 明示的に同意した場合だけ表情映像を保存
- 表情を大きく、表示コンテンツを小窓にしたリアクション映像の再生・書き出し
- IndexedDBの端末内ライブラリから再表示、共有、削除
- YouTube記録は映像を複製せず、URL・動画ID・分析値だけを端末内保存
- 旧版の `rear_blob`（外カメ動画）を新しい表示コンテンツとして読み込む非破壊の互換処理

## プライバシー

- 選択した端末内ファイル、内カメ映像、分析値は外部へ送信しません。
- YouTube選択時は公式プレイヤーで動画を再生するためYouTubeへ接続します。カメラ映像と分析値はYouTubeへ送信しません。
- 内カメ映像は、利用者が「表情映像も端末内に保存する」を選んだ場合だけ記録します。
- 選ばない場合、内カメ映像はリアルタイム解析後に破棄され、数値だけが残ります。
- 記録終了後のデータは同じブラウザのIndexedDBへ保存されます。
- ブラウザのサイトデータ削除、プライベートブラウズ、端末の容量整理によりライブラリが消える場合があります。重要な記録は共有またはファイル保存も行ってください。
- MediaPipeのJavaScriptとモデルファイルを読み込むため、初回にGoogle／jsDelivrのCDNへ接続します。選択コンテンツやカメラ映像は送信しません。

## YouTube共有から起動

対応ブラウザでViewPulseをPWAとしてインストールすると、YouTubeなどの共有メニューにViewPulseが表示されます。共有されたYouTube URLは開始画面へ自動入力されます。Web Share TargetはOS・ブラウザによって対応状況が異なり、すべてのiPhone・iPadでの表示は保証されません。

Netflixなど、公式の一般向け埋め込みプレイヤーを提供していない動画サービスには対応していません。

## 対応端末

カメラは内カメ1台だけを使うため、前後カメラ同時利用に由来する旧版のiPhone・iPad制限はありません。カメラとMediaRecorderに対応した主要ブラウザで利用できます。カメラ利用にはHTTPSまたはlocalhostが必要です。

ヒートマップは専用アイトラッカーではなく、内カメから推定したおおまかな視線です。研究利用では専用機器や自己報告との併用を推奨します。

## 起動

```powershell
node local-server.mjs
```

PCでは `http://127.0.0.1:8765`、スマートフォンではHTTPSで公開したURLを開きます。

## 検証

```powershell
node --check app.js
node --check local-server.mjs
node verify.mjs
```
